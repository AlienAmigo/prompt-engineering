## Описание

Фраза «Act as a senior React Native architect...» — это пример **ролевого промптинга** (Role Prompting), мощной техники, позволяющей направить ИИ на выдачу более качественных, структурированных и профессиональных ответов.

Вместо простого запроса «напиши компонент» вы задаёте модельке **роль, уровень экспертизы и контекст**, что активирует соответствующие «знания» из её обучающих данных.

---

## Что такое техника «Act as a senior...»?

Вы говорите модели:

> _«Ты — старший архитектор. Ты отвечаешь за масштабируемость, производительность и поддержку кода»._

Это не просто формальность — это способ **контекстуализировать запрос** и получить ответ, близкий к реальной практике senior-специалиста.

---

## Почему это работает эффективнее?

LLM обучены на огромных объёмах технической документации, форумов и кода. Упоминание роли:

1. ✅ Активирует **архитектурное мышление**
2. ✅ Заставляет учитывать **безопасность, доступность, производительность**
3. ✅ Делает ответ **структурированным и обоснованным**

Например, «архитектор» предложит использовать `React.memo`, `useCallback`, RTK Query или Compound Components — не потому что вы попросили, а потому что это логично для его роли.

## Реальные примеры промптов для Deepseek

### Пример 1: Создание нового экрана

```text
Act as a senior React Native architect.

Я работаю над проектом на React Native + TypeScript + Redux Toolkit.

**Контекст:**
- Версия RN: 0.73
- Структура: feature-first (папки по фичам)
- Есть `authSlice`, `cartSlice`
- Используем RTK Query для API

**Задача:** Создать экран "История заказов".

**Детали:**
- Список заказов с датой, суммой, статусом
- Данные с `/orders/history`
- Pull-to-refresh, скелетоны при загрузке
- Каждый заказ кликабелен

**Что нужно:**
1. Предложи структуру папок: `features/orders/history/...`
2. Напиши slice или предложи использовать RTK Query
3. Создай компонент экрана с типизацией
4. Обработай loading/error/stale состояния

Комментарии в коде — на английском, объяснения — на русском.
Если не хватает данных — уточни.
```

### Пример 2: Рефакторинг компонента

```text
Act as a senior React Native architect specializing in performance.

**Контекст:** Проект на RN 0.72, TypeScript, Redux.

**Проблема:** Компонент `ProductList` тормозит при скролле.

**Текущий код:**

[ВСТАВЬ СЮДА ТЕКУЩИЙ КОД `ProductList` (TSX), который нужно оптимизировать]

**Задача:** Оптимизировать производительность.

**Что нужно:**
1. Найди причины лишних ререндеров
2. Перепиши с React.memo, useCallback, useMemo
3. Проверь FlatList — ключи, getItemLayout, windowSize
4. Покажи оптимизированную версию
5. Объясни каждое изменение на русском.

---

### Пример 3: Исправление бага

```text
Act as a senior React Native architect.

**Контекст:** RN + TypeScript + Redux Persist + React Navigation v6

**Баг:** После логаута данные предыдущего пользователя остаются в сторе (например, корзина).

**Текущая логика:**
- При логауте вызываем `dispatch(resetStore())`
- Но Redux Persist частично сохраняет данные

**Задача:**
1. Объясни, почему так происходит
2. Предложи 2–3 решения
3. Рекомендуй лучшее под нашу архитектуру
4. Напиши код очистки стора и persist

Учти, что мы используем `persistReducer`.
```

## Секретные фишки для Deepseek на русском

💡 **Фишка 1: Смешивай языки**
Роль на английском: точнее активирует знания.
Задача на русском: понятнее для вас и команды.
✅ Работает лучше, чем полностью русский промпт.

💡 **Фишка 2: Проси объяснять на русском, код — на TS**
Добавляйте в конец:

"Комментарии в коде — на английском, объяснения — на русском."

💡 **Фишка 3: Проси задавать вопросы**
"Если не хватает информации — задай уточняющие вопросы, прежде чем писать код."

Deepseek часто действительно уточняет детали — как настоящий сеньор!

💡 **Фишка 4: Используй русские роли (альтернатива)
Модель понимает и такие формулировки:**

"Ты — опытный React Native разработчик"
"Представь, что ты тимлид"
"Действуй как архитектор мобильных приложений"
Но микс (английская роль + русская задача) даёт более предсказуемый результат.

## Полный шаблон для копирования (русскоязычный)

```text
Act as a senior React Native architect.

Я работаю над проектом на React Native + TypeScript + Redux Toolkit.
Вот контекст проекта:
- Версия RN: [вставить версию]
- Структура папок: [кратко описать, например: feature-based]
- Используемые слайсы: [authSlice, cartSlice и т.д.]
- API: [RTK Query / Axios / другой]

Задача: [подробно опишите задачу]

Технические требования:
1. Соблюдай структуру папок
2. Используй существующие слайсы, не создавай новые без необходимости
3. Код на TypeScript с правильной типизацией
4. Учти версию React Native [X.X]
5. Добавь обработку loading, error, success

Что нужно получить:
- [ ] Структура папок
- [ ] Архитектурное решение (slice, API, стор)
- [ ] Код компонента(ов)
- [ ] Объяснение логики на русском

Комментарии в коде — на английском.
Если что-то непонятно — задай вопросы.
```

## Чего НЕ делать

❌ Слишком абстрактно:

"Сделай dropdown."

❌ Без контекста:

"Act as a senior architect." — но не указано, что делать.

❌ Без указания стека:

Не ясно, TypeScript или JS, Redux или Zustand.

❌ Игнорирование версий:

RN 0.68 ≠ 0.73 — разный подход к CLI, Metro, New Architecture.

❌ Ожидание идеального кода без проверки:

Всегда проверяйте генерацию — возможны галлюцинации.

## Как передавать контекст, если нет загрузки файлов

Если вы используете API или чат без поддержки прикрепления файлов, вставьте ключевые фрагменты прямо в промпт:

```text
**Фрагменты проекта:**
- `package.json` (важные зависимости)
- `tsconfig.json` (настройки TS, path aliases)
- `navigation` (структура навигации)
- `store` (как устроен Redux/RTK Query)
- 1–2 ключевых компонента/хука, которые нужно расширить или отрефакторить

**Пример вставки:**
Файл: `features/orders/api/ordersApi.ts`
```ts
// ... важные места (типы, endpoints, baseQuery) ...
```

Файл: `features/orders/history/OrdersHistoryScreen.tsx`
```tsx
// ... текущая версия экрана или проблемный фрагмент ...
```

Дальше попроси:
1. Сначала перечислить, каких данных не хватает (если не хватает)
2. Затем предложить план изменений
3. Только после этого — дать код
```
